# The Pro-Crastinators Team 5 — Unity Project

## Getting Started

1. **Clone the repository:**
   ```bash
   git clone https://github.com/gaelxxl34/The-Pro-Crastinators-Team-5.git
   ```
2. **Open the project** in Unity Hub → "Open" → select the cloned folder.

## .gitignore — What We Ignore and Why

This project includes a `.gitignore` at the root. **Do NOT commit the files and folders listed below.** They are auto-generated by Unity or your OS and are different on every machine.

### Folders to NEVER commit

| Folder | Why it's ignored |
|---|---|
| `Library/` | Auto‑generated cache rebuilt by Unity when you open the project. **Large and machine‑specific.** |
| `Temp/` | Temporary files created while Unity is running. |
| `Obj/` | Intermediate build files. |
| `Build/` / `Builds/` | Compiled game builds — share builds separately, not via Git. |
| `Logs/` | Unity editor log files. |
| `UserSettings/` | Personal editor layout & preferences — different per user. |
| `MemoryCaptures/` | Memory profiler snapshots — can be huge. |
| `Recordings/` | Unity Recorder output files. |
| `.vs/` / `.vscode/` / `.idea/` | IDE/editor cache folders (Visual Studio, VS Code, Rider). |

### Files to NEVER commit

| Pattern | Why |
|---|---|
| `*.csproj` / `*.sln` / `*.slnx` / `*.suo` | Auto‑generated Visual Studio solution/project files — Unity recreates them. |
| `*.approj` | Unity project file — machine-specific. |
| `*.apk` / `*.aab` / `*.app` / `*.unitypackage` | Build outputs. |
| `*.pdb` / `*.mdb` / `*.opendb` / `*.VC.db` | Debug symbol / database files. |
| `.DS_Store` / `Thumbs.db` / `Desktop.ini` | OS junk files (macOS / Windows). |
| `*.7z` / `*.zip` / `*.rar` / `*.tar` / `*.gz` | Compressed archives — don't commit binaries. |
| `crashlytics-build.properties` | Auto-generated crash reporting config. |

### Files / Folders you SHOULD commit

| Path | Purpose |
|---|---|
| `Assets/` | All game assets: scripts, scenes, prefabs, sprites, audio, etc. |
| `Packages/manifest.json` | Declares which Unity packages the project uses. |
| `Packages/packages-lock.json` | Locks package versions for consistency across the team. |
| `ProjectSettings/` | Shared project settings (physics, quality, input, tags, etc.). |
| `.gitignore` | The ignore rules themselves. |
| `README.md` | This file. |

## For New Team Members — Setup Checklist

1. **Make sure you have the `.gitignore`** in your local repo root before your first commit. It is already in the repo, so cloning will give it to you automatically.
2. **Never force-add ignored files.** If Git is not tracking `Library/` or `UserSettings/`, that's correct — leave it that way.
3. **If you accidentally committed ignored files before**, run:
   ```bash
   git rm -r --cached Library/ Logs/ UserSettings/ Temp/ Obj/
   git commit -m "Remove previously tracked ignored files"
   git push
   ```
4. **Use the same Unity version** as the rest of the team (check `ProjectSettings/ProjectVersion.txt` for the exact version).

## Pushing Your Work

```bash
git add .
git commit -m "Describe your changes"
git push
```

Because the `.gitignore` is already set up, `git add .` will only stage the files that matter. You don't need to manually pick files.
