# The Pro-Crastinators Team 5

## Rules when pushing

Do **not** include these folders/files in your commits. They are auto-generated by Unity and will break things or bloat the repo:

- `Library/`
- `Temp/`
- `Obj/`
- `Build/` / `Builds/`
- `Logs/`
- `UserSettings/`
- `.vs/` / `.vscode/` / `.idea/`
- `*.csproj`, `*.sln`, `*.slnx`, `*.suo`, `*.approj`
- `.DS_Store`, `Thumbs.db`

The `.gitignore` in this repo already handles all of this. Just clone and push normally:

```
git add .
git commit -m "your message"
git push
```

## What to commit

Only these matter:

- `Assets/` — your scripts, scenes, prefabs, etc.
- `ProjectSettings/` — shared project settings
- `Packages/manifest.json` and `Packages/packages-lock.json`
